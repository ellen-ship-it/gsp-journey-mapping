Reactivation isn't fuzzy because you missed something — the underlying property date_last_reactivated is literally marked [WIP] in HubSpot. It's unfinished. Until it's locked, any win-back number is provisional. This is a real thing to flag, not a gap in your understanding.

Naming gotchas worth knowing before anyone writes a filter or query

  • "Sales Accepted Lead" ≠ its internal name. The SAL stage's internal value is salesqualifiedlead — reads like "qualified," labelled "accepted." Easy to grab the wrong thing in a report.
  • "Most Recent Demo Complete" has a misleading name. Internal name is first_demo_complete_contact, but it tracks the most recent demo, not the first. The genuine "first" version is a separate cloned property.
  • Account status has 5 values, not 3. active, trialing, cancelled, plus past_due and account_not_claimed. Reports that only count three will silently drop accounts.
  • Multiple "0–Planning / 1–Qualified / 2–Prove" stage sets exist across different pipelines. Always filter pipeline = New Business Pipeline or counts blend pipelines.
  • Use the (SEGMENT) versions of plan/status/date properties — they sync from the platform and are the reliable source of truth.
